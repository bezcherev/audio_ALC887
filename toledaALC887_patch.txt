The toledaALC887 patch configures OS X Realtek ALC887 onboard audio on Intel based motherboards with a bootable clean install of OS X.  The patched AppleHDA.kext only works with the codec the kext was edited for and replaces the native AppleHDA.kext.

toledaALC887_patch supports
1. Mountain Lion 10.8 - 10.8.4

Supported Audio-IDs (Audio_ID = layout-id)
1. Audio_ID: 1 - 5/6 port ALC887 onboard audio and AMD/Nvidia HDMI audio*
2. Audio_ID: 2 - 3 port ALC887 onboard audio and AMD/Nvidia HDMI audio*
3. Audio_ID: 3 - 3/5/6 port ALC887 onboard audio and HD3K/HD4K/AMD/Nvidia HDMI audio*
Audio_IDs: 1 and 2 support analog 5.1 surround sound, 3 does not
Audio_IDs: 1, 2 and 3 require HDMI audio dsdt edits for HDMI audio.

Requirements
1. Native S/L/E/AppleHDA.kext (use OS X combo update to restore native AppleHDA.kext)
2. Audio_ID injection enabled with one method below
2a. S/L/E/HDAEnabler1.kext or S/L/E/HDAEnabler2.kext
2b. dsdt/HDEF/layout-id=Audio_ID
2c. Clover with Config.plist/PCI/HDAInjection=Audio_ID
3. Realtek ALC887 installed on motherboard.

Download
1. https://github.com/toleda/audio_ALC887
2. Select: Download ZIP

Two ALC887 versions supported
1. ALC887_v100302 Current - Sandy Bridge/6 Series motherboards and newer
1a. Use as is: Downloads/audio_ALC887-master
2. ALC887_v100202 Legacy - 5 Series motherboards and older
2a. Make the following changes: Downloads/audio_ALC887-master
2b. Delete 887.zip
2c. Inflate 887_v100202.zip
2d. Delete 887_v100202.zip
2e. Rename 887_v100202 to 887
2f. Compress 887
2g. Delete 887 (887.zip remains)
3. ALC887_v100102 Not supported, use Interim AppleHDA.kext

Recommendation
1. Archive Desktop/audio_ALC887 folder as audio_ALC887_v10.x.x before patching a new AppleHDA.kext.  Patch must be run after every Software Update/Combo Update.

ALC887 Patch Procedure
Open Applications/Utilities/Terminal
One at a time, copy each command below, Paste into the Terminal window and press Enter
The ALC887 Patch makes a backup of the original AppleHDA.kext (AppleHDA-orig.kext) and a backup of the installed patched ALC887 AppleHDA.kext (AppleHDA.kext) available in Desktop/audio_ALC887. Uncommented Terminal commands below.

I.	Patch preparation/Backup installed AppleHDA.kext/Inflate zip file

cp -R Downloads/audio_ALC887-master Desktop/audio_ALC887
cp -R /System/Library/Extensions/AppleHDA.kext Desktop/audio_ALC887/AppleHDA-orig.kext
cd Desktop/audio_ALC887
unzip 887.zip
cd 887

II.	Install ALC887 layouts, pathmaps and pin configurations

sudo install -m 644 -o root -g wheel Info.plist /System/Library/Extensions/AppleHDA.kext/Contents/Plugins/AppleHDAHardwareConfigDriver.kext/Contents/Info.plist

sudo rm -R /System/Library/Extensions/AppleHDA.kext/Contents/Resources/layout*

sudo install -m 644 -o root -g wheel Platforms.xml.zlib layout1.xml.zlib layout2.xml.zlib layout3.xml.zlib  /System/Library/Extensions/AppleHDA.kext/Contents/Resources

III.	Install codec binary patch  
1. For ML 10.8 - 10.8.4:

sudo perl -pi -e 's|\xff\x87\xec\x1a\x0f\x8f\x53\x01\x00\x00|\x87\x08\xec\x10\x0f\x84\x2a\x01\x00\x00|g' /System/Library/Extensions/AppleHDA.kext/Contents/MacOS/AppleHDA

sudo perl -pi -e 's|\xff\x87\xec\x1a\x0f\x8f\x2f\x01\x00\x00|\x87\x08\xec\x10\x0f\x84\x06\x01\x00\x00|g' /System/Library/Extensions/AppleHDA.kext/Contents/MacOS/AppleHDA

IV.	Fix permissions and backup Patched ALC887 AppleHDA.kext

sudo chown -R root:wheel /System/Library/Extensions/AppleHDA.kext
sudo touch /System/Library/Extensions

cd ~
rm -R Desktop/audio_ALC887/887
cp -R /System/Library/Extensions/AppleHDA.kext Desktop/audio_ALC887/AppleHDA.kext

exit

V. 	Restart

VI.	ALC887 audio verification

1. Open System Preferences/Sound/Output and Select Internal Speakers
2. Connect speakers or headphones to the Green motherboard audio port
3. Open iTunes, verify output device is Computer, select a song and Press Play

VII	Troubleshooting

1. Verify S/L/E/AppleHDA.kext installed and version
2. Verify IOReg/HDEDF/Layout-id is 1, 2 or 3
3. Verify permissions: run Disk Utility/Select system disk/Repair Permissions and Restart

VIII.	Problem Reporting

1. Motherboard/BIOS version/Graphics/OS X version
2. Guide used, steps completed, problem description
2. Copy of IOReg
